## CSV exports

### Contributors CSV

Export complete contributor statistics to CSV format.

**Location:** `lib/analysis.ts`

```typescript
export function serializeContributorsCSV(contributors: ContributorStats[]): string {
  const headers = [
    'Name',
    'Email',
    'GitHub ID',
    'GitHub Login',
    'Commit Count',
    'Additions',
    'Deletions',
    'Net Lines',
    'First Commit',
    'Last Commit',
    'Active Days',
    'Has Merges'
  ];

  const rows = contributors.map(c => [
    escapeCSV(c.name),
    escapeCSV(c.email),
    c.githubId?.toString() || '',
    c.githubLogin || '',
    c.commitCount.toString(),
    c.additions.toString(),
    c.deletions.toString(),
    c.netLines.toString(),
    c.firstCommitDate || '',
    c.lastCommitDate || '',
    c.activeDays.toString(),
    c.isMergeCommitter ? 'Yes' : 'No'
  ]);

  return [headers.join(','), ...rows.map(r => r.join(','))].join('\n');
}
```

### CSV escaping

Proper escaping ensures data integrity:

```typescript
function escapeCSV(value: string | null | undefined): string {
  if (!value) return '';
  
  // Check if value needs quoting (contains comma, quote, or newline)
  const needsQuotes = /[,"\n\r]/.test(value);
  
  if (needsQuotes) {
    // Escape quotes by doubling them
    return `"${value.replace(/"/g, '""')}"`;
  }
  
  return value;
}
```

<Info>
CSV escaping follows RFC 4180 standards for maximum compatibility with spreadsheet applications.
</Info>

**Examples:**

| Input | Output |
|-------|--------|
| `John Doe` | `John Doe` |
| `Doe, John` | `"Doe, John"` |
| `John "The Boss" Doe` | `"John ""The Boss"" Doe"` |
| `Line 1\nLine 2` | `"Line 1\nLine 2"` |

## Text exports

### Commit messages export

Export all commit messages in a structured text format.

```typescript
export function serializeCommitMessages(messages: CommitMessage[]): string {
  return messages
    .map(m => `${m.sha.substring(0, 7)} | ${m.author} | ${m.date} | ${m.message}`)
    .join('\n');
}
```

**Schema:**

```typescript
export const CommitMessageSchema = z.object({
  sha: z.string(),
  author: z.string(),
  date: z.string(),
  message: z.string()  // First line only
});
```

**Example output:**

```text
a1b2c3d | johndoe | 2024-12-10T15:30:00Z | feat: add user authentication
e4f5g6h | janesmith | 2024-12-10T14:20:00Z | fix: resolve login bug
i7j8k9l | bobwilson | 2024-12-10T13:10:00Z | docs: update README
```

### Files by author export

Export file changes grouped by author.

```typescript
export function serializeFilesByAuthor(files: FileByAuthor[]): string {
  return files
    .map(f => `${f.author} | ${f.filename} | +${f.additions} -${f.deletions} (${f.changes})`)
    .join('\n');
}
```

**Schema:**

```typescript
export const FileByAuthorSchema = z.object({
  author: z.string(),
  filename: z.string(),
  additions: z.number(),
  deletions: z.number(),
  changes: z.number()
});
```

<Note>
Very large files (>10,000 changes) are excluded to prevent binary file noise.
</Note>

**Example output:**

```text
johndoe | src/auth.ts | +127 -34 (161)
johndoe | src/utils/validate.ts | +45 -12 (57)
janesmith | src/components/Login.tsx | +234 -89 (323)
bobwilson | README.md | +28 -5 (33)
```

### Merge commits export

Export all merge commit information.

```typescript
export function serializeMerges(merges: MergeCommit[]): string {
  return merges
    .map(m => `${m.sha.substring(0, 7)} | ${m.author} | ${m.date} | Parents: ${m.parentCount} | ${m.message}`)
    .join('\n');
}
```

**Schema:**

```typescript
export const MergeCommitSchema = z.object({
  sha: z.string(),
  author: z.string(),
  date: z.string(),
  message: z.string(),
  parentCount: z.number()  // Number of parent commits (>1 for merges)
});
```

**Example output:**

```text
m1n2o3p | johndoe | 2024-12-10T16:00:00Z | Parents: 2 | Merge pull request #123 from feature/auth
q4r5s6t | janesmith | 2024-12-09T18:30:00Z | Parents: 2 | Merge branch 'develop' into main
```

## Download implementation

### Client-side download

Implement CSV download in the browser:

```typescript app/dashboard/repo/[owner]/[repo]/branch/[branch]/page.tsx
const handleDownloadCSV = (type: "contributors" | "commits") => {
  if (analysisState.status !== "complete") {
    console.warn("Cannot download CSV: Analysis not complete");
    return;
  }

  try {
    // Get the appropriate CSV data
    const csv = type === "contributors"
      ? analysisState.data.exports.contributorsCSV
      : analysisState.data.exports.commitTimesText;

    if (!csv || typeof csv !== "string") {
      alert("CSV data is not available for download");
      return;
    }

    // Generate filename with timestamp
    const safeBranchName = decodedBranch.replace(/[^a-zA-Z0-9-_]/g, "_");
    const timestamp = new Date().toISOString().split("T")[0];
    const filename = `${owner}-${repo}-${safeBranchName}-${type}-${timestamp}.csv`;

    // Create blob and download
    const blob = new Blob([csv], { type: "text/csv;charset=utf-8;" });
    const url = URL.createObjectURL(blob);
    const a = document.createElement("a");
    a.href = url;
    a.download = filename;
    a.style.display = "none";
    document.body.appendChild(a);
    a.click();

    // Cleanup
    setTimeout(() => {
      document.body.removeChild(a);
      URL.revokeObjectURL(url);
    }, 100);
  } catch (error) {
    console.error("Failed to download CSV:", error);
    alert("Failed to download CSV. Please try again.");
  }
};
```

**Branch name sanitization:**

```typescript
const safeBranchName = decodedBranch.replace(/[^a-zA-Z0-9-_]/g, "_");
```

This ensures filenames are valid on all operating systems by:
- Removing special characters
- Keeping alphanumeric, hyphens, and underscores
- Replacing invalid characters with underscores
