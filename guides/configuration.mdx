## Environment variables

gitDash requires several environment variables to function properly. These are configured in your `.env.local` file for local development and in your hosting platform's environment settings for production.

<Warning>
Never commit `.env.local` to version control. This file contains sensitive credentials that should remain private.
</Warning>

### Required variables

All of these environment variables must be set for gitDash to work:

#### GITHUB_CLIENT_ID

Your GitHub OAuth App Client ID.

```bash
GITHUB_CLIENT_ID=your_github_client_id_here
```

<Steps>
  <Step title="Create a GitHub OAuth App">
    Navigate to [GitHub Developer Settings](https://github.com/settings/developers) and click "New OAuth App".
  </Step>
  
  <Step title="Configure OAuth App">
    - **Application name**: gitDash (or your preferred name)
    - **Homepage URL**: `http://localhost:3000` (for development)
    - **Authorization callback URL**: `http://localhost:3000/api/auth/callback/github`
  </Step>
  
  <Step title="Copy Client ID">
    After creating the app, copy the Client ID and add it to your `.env.local` file.
  </Step>
</Steps>

#### GITHUB_CLIENT_SECRET

Your GitHub OAuth App Client Secret.

```bash
GITHUB_CLIENT_SECRET=your_github_client_secret_here
```

<Note>
Generate a new client secret from your GitHub OAuth App settings page. This value is only shown once, so save it immediately.
</Note>

#### AUTH_SECRET

A random secret string used for signing JWT tokens and encrypting session data.

```bash
AUTH_SECRET=your_random_secret_here
```

<Tip>
Generate a secure random secret using OpenSSL:

```bash
openssl rand -base64 32
```

This generates a cryptographically secure 32-byte random string.
</Tip>

<Warning>
The `AUTH_SECRET` must be a strong, random string. Never use a predictable value like "secret" or "password". A weak secret compromises the security of all user sessions.
</Warning>

#### NEXTAUTH_URL

The base URL of your application.

```bash
# For local development
NEXTAUTH_URL=http://localhost:3000

# For production
NEXTAUTH_URL=https://yourdomain.com
```

<Note>
This must match the domain where your application is hosted. For production deployments, update this to your actual domain.
</Note>

### Optional variables

#### OPENAI_API_KEY

OpenAI API key for AI Manager Report functionality.

```bash
OPENAI_API_KEY=your_openai_api_key_here
```

The AI Manager Report feature requires an OpenAI API key. Without this, the feature will gracefully fall back with a message indicating the API key is not configured.

<Steps>
  <Step title="Get API key">
    Visit [OpenAI Platform](https://platform.openai.com/api-keys) and create a new API key.
  </Step>
  
  <Step title="Add to environment">
    Copy the key and add it to your `.env.local` file or production environment variables.
  </Step>
</Steps>

## Complete .env.local example

Here's a complete example of a `.env.local` file:

```bash
# GitHub OAuth App Configuration
# Create a GitHub OAuth App at: https://github.com/settings/developers
# Set the Authorization callback URL to: http://localhost:3000/api/auth/callback/github
GITHUB_CLIENT_ID=your_github_client_id_here
GITHUB_CLIENT_SECRET=your_github_client_secret_here

# NextAuth.js Configuration
# Generate a random secret with: openssl rand -base64 32
AUTH_SECRET=your_random_secret_here

# Application URL
# For local development: http://localhost:3000
# For production: https://yourdomain.com
NEXTAUTH_URL=http://localhost:3000

# OpenAI Configuration
# Get your API key from: https://platform.openai.com/api-keys
OPENAI_API_KEY=your_openai_api_key_here
```

## Next.js configuration

gitDash uses a minimal Next.js configuration with image optimization for GitHub avatars.

### next.config.mjs

```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: "https",
        hostname: "avatars.githubusercontent.com",
      },
    ],
  },
};

export default nextConfig;
```

This configuration allows Next.js to optimize images from GitHub's avatar CDN.

## Environment-specific configuration

### Development

For local development:

1. Create `.env.local` in your project root
2. Add all required environment variables
3. Use `http://localhost:3000` for all URLs
4. Set OAuth callback URL to `http://localhost:3000/api/auth/callback/github`

```bash
GITHUB_CLIENT_ID=dev_client_id
GITHUB_CLIENT_SECRET=dev_client_secret
AUTH_SECRET=$(openssl rand -base64 32)
NEXTAUTH_URL=http://localhost:3000
OPENAI_API_KEY=sk-...
```

### Production

For production deployments:

1. Set environment variables in your hosting platform
2. Use your production domain for `NEXTAUTH_URL`
3. Update OAuth callback URL to match your domain
4. Ensure `AUTH_SECRET` is different from development

```bash
GITHUB_CLIENT_ID=prod_client_id
GITHUB_CLIENT_SECRET=prod_client_secret
AUTH_SECRET=production_random_secret_here
NEXTAUTH_URL=https://gitdash.yourdomain.com
OPENAI_API_KEY=sk-...
```

<Warning>
Always use different OAuth Apps for development and production. Never share credentials between environments.
</Warning>

## Validating configuration

The application validates all required environment variables at startup. If any are missing, you'll see clear error messages:

```
Error: GITHUB_CLIENT_ID environment variable is required
Error: GITHUB_CLIENT_SECRET environment variable is required
Error: AUTH_SECRET environment variable is required
```

This validation is defined in `lib/auth.ts:14-25` and runs before the application starts.

## GitHub OAuth scopes

gitDash requests the following OAuth scopes:

- **repo**: Full access to repositories (required for analyzing private repos)
- **read:user**: Read user profile data
- **user:email**: Read user email addresses

These scopes are configured in `lib/auth.ts:32-36` and are necessary for the application to function properly.

<Note>
Users are prompted to authorize these scopes when they first sign in. They can review and revoke access at any time from their [GitHub authorized applications](https://github.com/settings/applications) page.
</Note>

## Configuration troubleshooting

<Accordion title="Missing environment variables">
  If you see errors about missing environment variables:
  
  1. Verify `.env.local` exists in your project root
  2. Check that all required variables are present
  3. Ensure there are no typos in variable names
  4. Restart the development server after making changes
</Accordion>

<Accordion title="OAuth callback mismatch">
  If OAuth fails with a redirect URI mismatch:
  
  1. Check that `NEXTAUTH_URL` matches your application URL exactly
  2. Verify the OAuth callback URL in GitHub matches `{NEXTAUTH_URL}/api/auth/callback/github`
  3. Ensure protocol (http/https) matches between environments
  4. Include port number for local development (`:3000`)
</Accordion>

<Accordion title="Session errors">
  If you experience session-related errors:
  
  1. Verify `AUTH_SECRET` is set and is a strong random string
  2. Clear browser cookies and try again
  3. Ensure `AUTH_SECRET` hasn't changed (this invalidates all sessions)
  4. Check that the secret is at least 32 characters long
</Accordion>
