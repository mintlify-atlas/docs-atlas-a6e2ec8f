# Quick start

This guide walks you through setting up gitDash and running your first contribution analysis.

## Prerequisites

Before you begin, ensure you have:

- **Node.js 18+** installed on your machine
- A **GitHub account** with access to repositories you want to analyze
- **(Optional)** OpenAI API key for AI Manager Report feature

## Setup steps

<Steps>
  <Step title="Create GitHub OAuth App">
    Create a GitHub OAuth application to enable authentication:

    1. Go to [GitHub Developer Settings](https://github.com/settings/developers)
    2. Click **New OAuth App**
    3. Fill in the application details:
       - **Application name**: `gitDash` (or your preferred name)
       - **Homepage URL**: `http://localhost:3000`
       - **Authorization callback URL**: `http://localhost:3000/api/auth/callback/github`
    4. Click **Register application**
    5. Copy the **Client ID**
    6. Click **Generate a new client secret** and copy the secret

    <Warning>
      Keep your Client Secret secure. Never commit it to version control or share it publicly.
    </Warning>

    <Note>
      For production deployments, you'll need to create a separate OAuth app with your production URL as the callback.
    </Note>
  </Step>

  <Step title="Clone and install dependencies">
    Clone the gitDash repository and install dependencies:

    ```bash
    git clone <your-gitdash-repo-url>
    cd gitdash
    npm install
    ```

    This installs all required packages including Next.js, Auth.js, Octokit, and Chart.js.
  </Step>

  <Step title="Configure environment variables">
    Create a `.env.local` file in the root directory with your credentials:

    ```bash
    cp .env.example .env.local
    ```

    Edit `.env.local` and add your configuration:

    <CodeGroup>
    ```bash .env.local
    # GitHub OAuth App Configuration
    # From the OAuth app you created in Step 1
    GITHUB_CLIENT_ID=your_github_client_id_here
    GITHUB_CLIENT_SECRET=your_github_client_secret_here

    # NextAuth.js Configuration
    # Generate with: openssl rand -base64 32
    AUTH_SECRET=your_random_secret_here

    # Application URL
    NEXTAUTH_URL=http://localhost:3000

    # OpenAI Configuration (Optional)
    # Required only for AI Manager Report feature
    OPENAI_API_KEY=your_openai_api_key_here
    ```

    ```bash Generate AUTH_SECRET
    # Generate a secure random secret
    openssl rand -base64 32
    ```
    </CodeGroup>

    ### Required environment variables

    | Variable | Description | Example |
    |----------|-------------|----------|
    | `GITHUB_CLIENT_ID` | OAuth App Client ID from GitHub | `Iv1.a1b2c3d4e5f6g7h8` |
    | `GITHUB_CLIENT_SECRET` | OAuth App Client Secret from GitHub | `a1b2c3d4e5f6...` |
    | `AUTH_SECRET` | Random secret for JWT signing | Generate with openssl |
    | `NEXTAUTH_URL` | Your application URL | `http://localhost:3000` |

    ### Optional environment variables

    | Variable | Description | Required for |
    |----------|-------------|-------------|
    | `OPENAI_API_KEY` | OpenAI API key | AI Manager Report feature |

    <Warning>
      Never commit `.env.local` to version control. It's already in `.gitignore` by default.
    </Warning>
  </Step>

  <Step title="Run the development server">
    Start the Next.js development server:

    ```bash
    npm run dev
    ```

    The application will be available at [http://localhost:3000](http://localhost:3000).

    <Note>
      The first build may take a minute. Subsequent starts will be faster with Next.js caching.
    </Note>
  </Step>

  <Step title="Sign in and run your first analysis">
    1. Open [http://localhost:3000](http://localhost:3000) in your browser
    2. Click **Sign in with GitHub**
    3. Authorize the application when prompted
    4. Navigate to the dashboard at `/dashboard/analyze`
    5. Select a repository and branch
    6. Choose an analysis mode:
       - **Blame**: Most accurate for code ownership
       - **Commits**: Good for activity timelines
       - **GitHub API**: Fast metadata-only analysis
       - **Hybrid**: Complete analysis with all data sources
    7. Click **Start Analysis** and watch the progress

    <Note>
      For local git modes (blame, commits, hybrid), you'll need to provide the absolute path to a local clone of the repository.
    </Note>
  </Step>
</Steps>

## Troubleshooting

### Authentication fails

- Verify your `GITHUB_CLIENT_ID` and `GITHUB_CLIENT_SECRET` are correct
- Check that the callback URL matches exactly: `http://localhost:3000/api/auth/callback/github`
- Ensure `AUTH_SECRET` is set and is a strong random string
- Clear browser cookies and try again

### Build errors

- Confirm Node.js version is 18 or higher: `node --version`
- Delete `node_modules` and reinstall: `rm -rf node_modules && npm install`
- Run type checking to identify issues: `npm run type-check`

### Rate limiting

- GitHub API has a limit of 5,000 authenticated requests per hour
- Large repositories may require multiple requests
- Wait for the rate limit to reset (shown in error message)
- Consider using local git modes (blame, commits) which don't count against API limits
