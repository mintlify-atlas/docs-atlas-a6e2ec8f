## Endpoints

**Standard analysis:**
```
POST /api/github/analyze
```

**Real-time streaming analysis:**
```
POST /api/github/analyze/stream
```

Analyzes repository contributions using various methods. Supports multiple analysis modes including blame-based attribution, commit statistics, GitHub API metadata, and hybrid analysis.

<Note>
For real-time progress updates during analysis, use the `/api/github/analyze/stream` endpoint which returns Server-Sent Events (SSE). This is recommended for large repositories where analysis may take several minutes.
</Note>

## Authentication

Requires valid NextAuth session with GitHub OAuth access token.

## Analysis Modes

The endpoint supports five different modes:

1. **`blame`** - True line-level code ownership (requires local repository)
2. **`commits`** - Commit activity analysis with `--no-merges` (requires local repository)
3. **`github-api`** - GitHub metadata only (PRs, issues, contributors)
4. **`hybrid`** - Combines all of the above
5. **Legacy mode** (default) - Original GitHub API commit analysis

## Example Requests

### Blame Mode

```bash cURL
curl -X POST "https://your-domain.com/api/github/analyze" \
  -H "Content-Type: application/json" \
  -H "Cookie: next-auth.session-token=YOUR_SESSION_TOKEN" \
  -d '{
    "mode": "blame",
    "repoPath": "/path/to/repository"
  }'
```

```javascript JavaScript
const response = await fetch('/api/github/analyze', {
  method: 'POST',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    mode: 'blame',
    repoPath: '/path/to/repository',
    blameOptions: {
      ignoreWhitespace: true,
      detectMoves: true
    }
  })
});

const data = await response.json();
```

### Legacy Mode

```bash cURL
curl -X POST "https://your-domain.com/api/github/analyze" \
  -H "Content-Type: application/json" \
  -H "Cookie: next-auth.session-token=YOUR_SESSION_TOKEN" \
  -d '{
    "owner": "octocat",
    "repo": "hello-world",
    "branch": "main",
    "includeBots": false
  }'
```

```python Python
import requests

response = requests.post(
    'https://your-domain.com/api/github/analyze',
    json={
        'owner': 'octocat',
        'repo': 'hello-world',
        'branch': 'main',
        'includeBots': False
    },
    cookies={'next-auth.session-token': 'YOUR_SESSION_TOKEN'}
)

data = response.json()
print(f"Analyzed {data['metadata']['analyzedCommits']} commits")
```

## Streaming Analysis

For real-time progress updates, use the streaming variant:

### POST /api/github/analyze/stream

Returns Server-Sent Events (SSE) with progress updates during analysis.

**Request Body (Legacy Mode Streaming)**

<ParamField body="owner" type="string" required>
  Repository owner (username or organization)
</ParamField>

<ParamField body="repo" type="string" required>
  Repository name
</ParamField>

<ParamField body="branch" type="string" required>
  Branch name to analyze
</ParamField>

<ParamField body="since" type="string">
  Start date in ISO format
</ParamField>

<ParamField body="until" type="string">
  End date in ISO format
</ParamField>

<ParamField body="filterBots" type="boolean" default={true}>
  Filter out bot commits from analysis
</ParamField>

**SSE Event Types**

The streaming endpoint sends events in the following format:

```json
data: {"type":"progress","percent":50,"message":"Processing commits..."}
```

```json
data: {"type":"complete","result":{...analysis data...}}
```

```json
data: {"type":"error","message":"Error occurred","code":"ERROR_CODE"}
```

**Event Fields**

- `type`: Event type (`progress`, `complete`, or `error`)
- `percent`: Progress percentage (0-100) for progress events
- `message`: Human-readable status message
- `result`: Complete analysis result (only in `complete` events)
- `code`: Error code (only in `error` events)

**Example Usage**

```javascript
const response = await fetch('/api/github/analyze/stream', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    owner: 'username',
    repo: 'repository',
    branch: 'main',
    filterBots: true
  })
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const chunk = decoder.decode(value);
  const lines = chunk.split('\n');
  
  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const event = JSON.parse(line.slice(6));
      
      if (event.type === 'progress') {
        console.log(`Progress: ${event.percent}% - ${event.message}`);
      } else if (event.type === 'complete') {
        console.log('Analysis complete:', event.result);
      } else if (event.type === 'error') {
        console.error('Error:', event.message);
      }
    }
  }
}
```

## Performance Notes

- **Legacy mode**: Fetches up to 5,000 commits per request
- **Streaming**: Provides real-time progress for long-running operations
- **Blame mode**: Processes files concurrently for performance
- **Hybrid mode**: Runs all analyses in parallel
- Large repositories may take several minutes to analyze
- Streaming endpoint has a 5-minute timeout (maxDuration: 300)
