## Overview

Advanced analysis provides comprehensive repository insights including timeline data, user contribution metrics, language distribution, collaboration patterns, and commit message analysis.

**Two endpoints available:**

1. **POST /api/github/analyze/advanced** - Standard JSON response with 24-hour caching
2. **POST /api/github/analyze/advanced/stream** - Server-Sent Events (SSE) with real-time progress updates

---

## POST /api/github/analyze/advanced

### Endpoint

```
POST /api/github/analyze/advanced
```

Performs comprehensive analysis with automatic 24-hour caching for improved performance.

Requires valid NextAuth session with GitHub OAuth access token.

### Caching Behavior

- **Cache duration:** 24 hours (86,400 seconds)
- **Cache key:** Based on `owner`, `repo`, `branch`, `since`, `until`
- **Cache tags:** `analysis-{owner}-{repo}-{branch}`
- **Revalidation:** Automatic after TTL expires

### Performance

- **Cached responses:** < 100ms
- **Fresh analysis:** 5-30 seconds depending on repository size
- **API calls:** 1 + ceil(commits/100) GitHub API requests
- **Max commits:** 5,000 per request

## POST /api/github/analyze/advanced/stream

### Endpoint

```
POST /api/github/analyze/advanced/stream
```

Streams analysis progress via Server-Sent Events (SSE) with real-time updates.

### Authentication

Requires valid NextAuth session with GitHub OAuth access token.

### Request Body

Same as the standard endpoint, with one additional parameter:

<ParamField body="offset" type="number" default="0">
  Pagination offset for large repositories with 10,000+ commits
</ParamField>

### Server-Sent Events (SSE)

The endpoint streams three types of events:

#### Progress Event

```json
{
  "type": "progress",
  "message": "Fetching commits...",
  "percent": 45
}
```

<ResponseField name="type" type="string">
  Always `"progress"`
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable progress description
</ResponseField>

<ResponseField name="percent" type="number">
  Progress percentage (0-100)
</ResponseField>

#### Complete Event

```json
{
  "type": "complete",
  "data": { /* AdvancedAnalysisResponse */ },
  "hasMore": false,
  "nextOffset": 0
}
```

<ResponseField name="type" type="string">
  Always `"complete"`
</ResponseField>

<ResponseField name="data" type="object">
  Full analysis response (same structure as standard endpoint)
</ResponseField>

<ResponseField name="hasMore" type="boolean">
  Whether more commits are available beyond the 5,000 limit
</ResponseField>

<ResponseField name="nextOffset" type="number">
  Offset to use for next request if `hasMore` is true
</ResponseField>

#### Error Event

```json
{
  "type": "error",
  "message": "GitHub API rate limit low (50 remaining). Please try again later.",
  "code": "RATE_LIMIT_LOW"
}
```

<ResponseField name="type" type="string">
  Always `"error"`
</ResponseField>

<ResponseField name="message" type="string">
  Error description
</ResponseField>

<ResponseField name="code" type="string">
  Error code for programmatic handling
</ResponseField>

### Example Request (SSE)

```javascript JavaScript
const response = await fetch('/api/github/analyze/advanced/stream', {
  method: 'POST',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    owner: 'facebook',
    repo: 'react',
    branch: 'main'
  })
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const text = decoder.decode(value);
  const lines = text.split('\n');
  
  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const event = JSON.parse(line.slice(6));
      
      if (event.type === 'progress') {
        console.log(`${event.percent}%: ${event.message}`);
      } else if (event.type === 'complete') {
        console.log('Analysis complete!', event.data);
      } else if (event.type === 'error') {
        console.error('Error:', event.message);
      }
    }
  }
}
```

```bash cURL (SSE)
curl -N -X POST "https://your-domain.com/api/github/analyze/advanced/stream" \
  -H "Content-Type: application/json" \
  -H "Cookie: next-auth.session-token=YOUR_SESSION_TOKEN" \
  -d '{
    "owner": "facebook",
    "repo": "react",
    "branch": "main"
  }'
```

### Progress Updates

Progress events are sent at the following stages:

- **0%** - Starting analysis
- **0-80%** - Fetching commits (updates every 100 commits)
- **82%** - Processing timeline data
- **85%** - Aggregating weekly metrics
- **90%** - Extracting user contributions
- **95%** - Generating insights
- **98%** - Analyzing commit messages
- **100%** - Complete

### Response Headers

```
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
X-Accel-Buffering: no
```
