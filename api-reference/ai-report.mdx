## Endpoint

```
POST /api/ai/manager-report
```

Generates comprehensive, AI-powered insights and recommendations for engineering managers based on repository analysis data.

## Authentication

Requires valid NextAuth session with GitHub OAuth access token.

## Configuration Requirements

### OpenAI API Key

This endpoint requires `OPENAI_API_KEY` to be configured in your environment variables:

```bash .env.local
OPENAI_API_KEY=sk-...
```

If not configured, the endpoint returns a `503 Service Unavailable` error with a fallback flag.

## Request Body

The request body should be the complete response from the Advanced Analysis endpoint.

## Example Request

```bash cURL
curl -X POST "https://your-domain.com/api/ai/manager-report" \
  -H "Content-Type: application/json" \
  -H "Cookie: next-auth.session-token=YOUR_SESSION_TOKEN" \
  -d '{
    "timeline": {
      "totalCommits": 523,
      "totalAdditions": 45230,
      "totalDeletions": 12450,
      "users": [...]
    },
    "insights": {
      "mostActiveDay": {...},
      "languageDistribution": [...]
    },
    "commitMessageAnalysis": {
      "statistics": {...}
    },
    "metadata": {
      "analyzedCommits": 523,
      "dateRange": {
        "start": "2024-01-01T00:00:00Z",
        "end": "2024-03-03T23:59:59Z"
      }
    }
  }'
```

```javascript JavaScript
// First, get the advanced analysis
const analysisResponse = await fetch('/api/github/analyze/advanced', {
  method: 'POST',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    owner: 'facebook',
    repo: 'react',
    branch: 'main'
  })
});

const analysisData = await analysisResponse.json();

// Then, generate AI insights
const aiResponse = await fetch('/api/ai/manager-report', {
  method: 'POST',
  credentials: 'include',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(analysisData)
});

const report = await aiResponse.json();
console.log(report.insights.executiveSummary);
console.log('Team Health Score:', report.insights.teamHealth.score);
```

## Caching Behavior

### Cache Key Generation

From `route.ts:126-130`:

```typescript
function generateCacheKey(data: AdvancedAnalysisResponse): string {
  const { timeline, metadata } = data;
  return `ai_insights_${timeline.totalCommits}_${metadata.dateRange.start}_${metadata.dateRange.end}`;
}
```

### Cache Settings

- **Storage:** In-memory Map (consider Redis for production)
- **TTL:** 1 hour (3,600,000 milliseconds)
- **Cache key:** Based on commit count and date range
- **Invalidation:** Automatic after TTL expires

## Fallback Handling

All error responses include `"fallback": true`, indicating that the client should display traditional analysis instead of AI-generated insights.

## Analysis Prompt

The AI receives comprehensive data including:

- Total commits, contributors, lines changed
- Top contributors with detailed metrics
- Activity patterns (weekday vs weekend, most active day)
- File analysis (most edited files, collaboration patterns)
- **Commit message analysis:**
  - Message length statistics and distribution
  - Conventional commit compliance percentage
  - User writing style categories (verbose, minimalist, balanced)
  - Quality patterns (generic messages, single-char commits, empty messages)

From `route.ts:185-306`, the prompt instructs the AI to:

1. Analyze overall sentiment and professionalism
2. Assess message quality and usefulness
3. Evaluate adherence to conventional commit standards
4. Provide specific recommendations for improvement
5. Compare to industry best practices
6. Assess impact on collaboration and maintainability

## Use Cases

- **Performance reviews:** Data-driven insights for quarterly reviews
- **Team health monitoring:** Track collaboration patterns and work-life balance
- **Process improvement:** Identify commit message quality issues and best practices
- **Knowledge management:** Detect knowledge silos and single points of failure
- **Strategic planning:** Predictive insights for future trends
